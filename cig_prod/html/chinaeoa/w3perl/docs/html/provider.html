HOW TO USE FTP TO SET UP THE W3PERL SCRIPT ON YOUR WEB SITE



  • 1st STAGE : Pre-requis

    - Things to check :

    • First of all, do you have access to your log files ?
    • Do you know the path to your log files ?
    • Do you have cgi-bin rights ?

    If the answer is yes, then you can use this script.

    - The next thing to know is the PATH to your account.
    - If you do not know what the path is, ask your ISP. It's the most basic piece of information that you need for setting up CGI scripts.
    - Now make two directories called w3perl in your web. Make one for your cgi-bin and one for your root directory.

  • 2nd STAGE : Install the scripts

    - Next you need the w3perl.tar file. Download and unzip this.
    - What you have here may confuse you because various files are scattered through a number of directories, but essentially find all the .pl files in the collection and put them into your cgi-bin's w3perl directory.
    - Here are the files that I use in my cgi-bin and I assume that if you have them then yours will also work:

    • cron-agent.pl
    • cron-day.pl
    • cron-error.pl
    • cron-hour.pl
    • cron.inc.pl
    • cron-month.pl
    • cron-pages.pl
    • cron-history.pl
    • cron-refer.pl
    • cron-session.pl
    • cron-url.pl
    • cron-w3perl.pl
    • cron-week.pl
    • config-unix.pl
    • fixerlpath.pl
    • install.pl
    • libw3perl.pl
    • confstat.pl
    • delconf.pl
    • init.pl

    - Upload them in ASCII not Binary mode.
    - Set them all to CHMOD 755

  • 3rd STAGE : Install the resource files

    - In the w3perl directory in your root directory load in the following directories:

    • admin
    • config
    • docs
    • resources
    • tools

    With all their contents!
    - Set them all at CHMOD 777
    - This is something a blanket approach because you will find files that are duplicated in different languages and some functional files that are for people using NT servers rather than UNIX and so on.
    - But for simplicity's sake put them all in. You can rationalise once you have everything working.

  • 4th STAGE : Graphic tool installation

    - If you want a Graphical display of your statistics you need a programme called FLY
    - Ask your ISP what Operation System they are using and then go to http://www.unimelb.edu.au/fly/fly.html
    - Download the READY COMPILED BINARY for that operating system. If they do not have one for your particular operation system then you can try asking the people there or you can get hold of the SOURCE file and give it to your ISP and ask them to do you one.
    - If you can get hold of a READY COMPILED BINARY all you need do is download the zipped package,
    - Unzip and ignore everything in the package apart from one file called FLY.
    - Place this in your cgi-bin and CHMOD it 775
    - Remember where you put it because you will have to fill that in during the configuration process.
    - If you cannot get hold of a copy, the programme will still work except you won't be able to see some nice histograms and pie charts.

  • 5th STAGE : Configure path

    - Now open install.pl in notepad or use the WSFTP edit function.
    - Check your Perl path is right (first line)
    - Set $pathcgi = "/www/yourpath/cgi-bin/w3perl"; and $pathw3perl = "/www/yourpath/w3perl";
    - Replace in cgi-bin/w3perl

  • 6th STAGE : Build configuration file

    - Now open up your browser and go to http://your_url/w3perl/admin
    - Click on Make configuration file and follow the procedures outlined. By filling in the details you will create a file called config.pl
    - Once created, look for this file in http://your_url/w3perl/config and move it to the w3perl directory in your cgi_bin,
    - CHMOD to 755

  • 7th STAGE : Ready

    - Return to your browser. Go to admin and run Customization
    - This will set all the paths in all the files.
    - You should now be able to analyse your logs.
    - In admin go through the FIRST USE links that will start your statistics from zero.
    - Then run the incremental statistics function every week to accumulate your statistics.
    - In order to read them go to http://your_url/w3perl/stats
