Niveau de précision
Les résultats fournis par l'application dépendront du niveau de pr&ecute;cision que vous choisirer. Plus la précision sera élevée, plus le temps de calcul sera important.
Le niveau de précision est contenu dans la variable $precision. Vous pouvez augmenter sa valeur en cours du temps mais pas la dimninuer sans réinitialiser toutes les stats.
Les machines utilisant des systèmes d'exploitation ne gérant pas les noms de fichiers longs ne doivent pas utiliser le mode de précision 4 sauf si vous utilisez bien sur des programmes qui contournent ses limitations.

La plupart des serveurs utilisent le mode 3, n'utilisez le mode 4 que pour des sites avec un traffic faible.
Les scripts optionnels (error, agent, refer, session et url) ont des sorties indépendante de la précision choisie.

Simple
($precision = 1) 		Seul cron-inc est nécessaire dans ce mode.
Auncune information sur l'évolution temporelle n'est donnée.
la sortie ne consiste qu'en une page unique avec les informations suivantes :
  • Les pages les plus lues.
  • Les sites ayant accédés le plus votre site
  • Le nombre de sites, de pays et de requêtes

Minimal
($precision = 2)
  • Stats sur les heures : seuls les accès sur les sites par heures et les meilleures pages sont affichés.
  • Stats sur les jours : seul le graphe sur les requêtes est produit.
  • Stats sur les semaines : seul le graphe sur les requêtes est produit.
  • Stats sur les mois : seul le graphe sur les requêtes est produit ainsi que les résumés pour chaque mois.
  • Stats sur les pages : seuls les pages lues et les sites sont affichés. Auncun détail sur les sites.
