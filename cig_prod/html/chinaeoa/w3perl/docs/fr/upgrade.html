Mise à jour de l'application
Si vous utilisez une version antérieure à 2.50, je vous recommande de tout réinstaller car beaucoup de choses ont changées depuis. Certains fichiers ont changés de structure dans leur méthode de sauvegarde de données.

Procédure Certaines mises à jour nécessitent de réinitialiser vos stats car certaines structures de données incrémentales ont été modifiées.

  1. Effectuez une copie de vos fichiers sensibles

    Sauvegardez vos fichier de configuration, ainsi que le fichier d'install.pl et le script cron-w3perl.pl

  2. Extraction

    Décompressez l'application qui va remplacer les scripts existants.

  3. Configuration

    • Remplissez le nouveau fichier de configuration config.pl avec l'aide de votre ancien fichier config/config.pl. Copiez, et remplissez les nouveaux champs. Il est possible d'utiliser l'interface d'adminstration pour générer le fichier de configuration si vous utilisez une version récente de w3perl.
    • Copier/coller $pathcgi, $pathw3perl dans le nouveau script d'installation ainsi que la première ligne qui indique le chemin où se situe Perl.
    • Copier/coller les valeurs que vous avez donner dans cron-w3perl.pl dans le nouveau script.

  4. Installation

    Lorsque tout est prêt, lancer install.pl .... maintenant vous êtes prêt à utiliser la dernière version de w3perl

Si vous désirez garder une copie de vos vieilles stats avant de les effacer, ajoutez le vieux répertoire de w3perl dans @excluderepert au niveau de config.pl (ex : @excluderepert = ('/w3perl.old/');)

Réinitialisation La plupart des scripts tourne de manière incrémentale. Si vous avez besoin de réinitialiser toutes vos stats, vous devrez lancer cron-w3perl.pl avec l'option -u. Cela détruira tous vos fichiers de stats and relancera les scripts pour recalculer vos stats.
Si vous ne désirez que réinitialiser que certains scripts, utilisez l'option -b qui effacera les valeurs incrémentales stockées associées au script. (voir les options disponibles avec l'option -h).
