Compresser le fichier de log
Le but de squeezelog est de compresser votre énorme fichier de log pour gagner de la place sur votre disque.
Chaque premier jour de chaque mois, squeezelog doit être lancé dans une crontab pour couper et compresser le fichier de log courant. Il ne peut être lancer qu'un super utilisateur car les fichiers de log sont le plus souvent des fichiers systèmes.

(Ex : 01 00 1 * * /usr/bin/sh /norfolk/www/w3perl/squeezelog)

Configurer squeezelog :

  • LOGDIR est le chemin où sont situés les fichier de log
  • LOG est le nom de fichier de log
  • Vous devez donner le chemin complet pour gzip (Ex : /opt/bin/gzip)
Modifier squeezelog :
  • Si vous avez un fichier de log d'erreur, vous pouvez aussi le compresser (effacez les commenataires dans le fichier squeezelog)
  • Vous pouvez aussi relancer le daemon httpd
    (voir la dernière ligne en commentaire, '/home/local/WWW/server_root/httpd-pid' est le fichier où se trouve le pid du daemon httpd).
