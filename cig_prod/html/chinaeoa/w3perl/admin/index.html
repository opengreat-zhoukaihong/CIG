W3Perl administration
Should be located in a restricted area
(Version 2.72)

Config files
Before using this interface, you should have edit and run the install.pl script.
Your first step will be to create a configuration file. You can have several configuration files for one server or one configuration file for each of your server. This file have everything to customize your stats.

Make configuration files

You could use as many configuration files as you need if you want to view different stats with the same logfile.
The config file produced by the script should be allowed to be written in your install directory, so give permission to read, write in /config/.

Modify configuration files

Use this part if you just need to change some values in your configuration files.
This part will load your current data and will allow you to change the values.

Delete configuration files

If you want to delete some of your configuration files, this is the right place.
Be careful as there is no way to restore them.

Launch
Launch scripts

If you don't have a telnet access or you are not allowed to have a crontab, you can launch the scripts via a HTML page. For each scripts, options are available.

Upgrade/Install

Upgrade package

If you want to upgrade to the latest release, please use this script which will detect any change in the configuration file. It works from version 2.69
Fix path scripts

If you have installed the package from a ftp client (without telnet) or you have upgraded w3perl from a previous version, you should run this script first to enable your cgi to run. It will fix path inside the scripts according to your setup.
