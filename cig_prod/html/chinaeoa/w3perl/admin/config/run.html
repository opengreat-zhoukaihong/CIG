W3Perl Run
If you can only run scripts via a HTML page (people without telnet access or/and crontab), use these links to run the scripts. On Unix, scripts have very restrictive permission (user nobody usually), so you should verify the scripts will be able to write output files in your directory.

If you want a fine tuning, select the 'option' you want to run. All option available will be displayed with their current values. Default scripts will be run as incremental, if you need to reset or initialize them, you'll need to use the option -b.

If the scripts take too long to complete, you'll receive a time out from the server. Don't worry, you'll have to use a shortest period of time (-d) or giving a more lower precision level (-p). Of course, the best way to run the stats is to get a crontab.
